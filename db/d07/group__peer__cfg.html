enum cert_policy_t

Certificate sending policy. This is also used for certificate requests when using this definition for the other peer. If it is CERT_NEVER_SEND, a certreq is omitted, otherwise its included.

Warning
These definitions must be the same as in pluto/starter, as they are sent over the stroke socket.
Enumerator
CERT_ALWAYS_SEND 

always send certificates, even when not requested

CERT_SEND_IF_ASKED 

send certificate upon cert request

CERT_NEVER_SEND 

never send a certificate, even when requested

enum unique_policy_t

keep existing IKE_SAs, close the new ones on connection attempt

Create a configuration object for IKE_AUTH and later.

name-string gets cloned, ID's not. Virtual IPs are used if they are != NULL. A any host means the virtual IP should be obtained from the other peer. Lifetimes are in seconds. To prevent to peers to start rekeying at the same time, a jitter may be specified. Rekeying of an SA starts at (rekeylifetime - random(0, jitter)).
