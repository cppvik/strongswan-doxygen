status_t(* esp_packet_t::decrypt)(esp_packet_t *this, esp_context_t *esp_context)

Authenticate and decrypt the packet. Also verifies the sequence number using the supplied ESP context and updates the anti-replay window.

Parameters
esp_contextESP context of corresponding inbound IPsec SA
Returns
- SUCCESS if successfully authenticated, decrypted and parsed
  • PARSE_ERROR if the length of the packet or the padding is invalid
  • VERIFY_ERROR if the sequence number verification failed
  • FAILED if the ICV (MAC) check or the actual decryption failed

status_t(* esp_packet_t::encrypt)(esp_packet_t *this, esp_context_t *esp_context, u_int32_t spi)

Encapsulate and encrypt the packet. The sequence number will be generated using the supplied ESP context.

Parameters
esp_contextESP context of corresponding outbound IPsec SA
spiSPI value to use, in network byte order
Returns
- SUCCESS if encrypted
  • FAILED if sequence number cycled or any of the cryptographic functions failed
  • NOT_FOUND if no suitable IV generator provided
