Detailed Description

TLS specific AEAD interface, includes padding.

As TLS uses sign-then-encrypt instead of the more modern encrypt-then-sign, we can't directly abstract traditional transforms using our aead_t interface. With traditional transforms, the AEAD operation has to manage padding, as the MAC is calculated over unpadded data.

bool(* tls_aead_t::encrypt)(tls_aead_t *this, tls_version_t version, tls_content_type_t type, u_int64_t seq, chunk_t *data)

Encrypt and sign a TLS record.

The plain data chunk gets freed on success, and the data chunk gets updated with a new allocation of the encrypted data. If next_iv is given, it must contain the IV for this operation. It gets updated to the IV for the next record.
