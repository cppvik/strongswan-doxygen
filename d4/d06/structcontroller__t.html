Detailed Description

The controller provides a simple interface to run actions.

The controller starts actions by creating jobs. It then tries to evaluate the result of the operation by listening on the bus.

Passing NULL as callback to the managers function calls them asynchronously. If a callback is specified, they are called synchronously. There is a default callback "controller_cb_empty" if you want to call a function synchronously, but don't need a callback.
