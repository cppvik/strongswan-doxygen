Detailed Description

A child_cfg_t defines the config template for a CHILD_SA.

After creation, proposals and traffic selectors may be added to the config. A child_cfg object is referenced multiple times, and is not thread save. Reading from the object is save, adding things is not allowed while other threads may access the object. A reference counter handles the number of references hold to this config.

See Also
peer_cfg_t to get an overview over the configurations.

linked_list_t*(* child_cfg_t::get_traffic_selectors)(child_cfg_t *this, bool local, linked_list_t *supplied, linked_list_t *hosts)

Get a list of traffic selectors to use for the CHILD_SA.

The config contains two set of traffic selectors, one for the local side, one for the remote side. If a list with traffic selectors is supplied, these are used to narrow down the traffic selector list to the greatest common divisor. Some traffic selector may be "dymamic", meaning they are narrowed down to a specific address (host-to-host or virtual-IP setups). Use the "host" parameter to narrow such traffic selectors to that address. Resulted list and its traffic selectors must be destroyed after use.

Parameters
localTRUE for TS on local side, FALSE for remote
suppliedlist with TS to select from, or NULL
hostsaddresses to use for narrowing "dynamic" TS', host_t
Returns
list containing the traffic selectors
