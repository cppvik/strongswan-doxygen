Detailed Description

Interface for a (U)SIM card (used as EAP client).

The SIM card completes triplets/quintuplets requested in a challenge received from the server. An implementation supporting only one of SIM/AKA authentication may implement the other methods with return_false()/return NOT_SUPPORTED/NULL.

status_t(* simaka_card_t::get_quintuplet)(simaka_card_t *this, identification_t *id, char rand[AKA_RAND_LEN], char autn[AKA_AUTN_LEN], char ck[AKA_CK_LEN], char ik[AKA_IK_LEN], char res[AKA_RES_MAX], int *res_len)

Calculate CK/IK/RES from RAND/AUTN for AKA authentication.

If the received sequence number (in autn) is out of sync, INVALID_STATE is returned. The RES value is the only one with variable length. Pass a buffer of at least AKA_RES_MAX, the actual number of bytes is written to the res_len value. While the standard would allow any bit length between 32 and 128 bits, we support only full bytes for now.

Parameters
idpermanent identity to request quintuplet for
randrandom value rand
autnauthentication token autn
ckbuffer receiving encryption key ck
ikbuffer receiving integrity key ik
resbuffer receiving authentication result res
res_lennubmer of bytes written to res buffer
Returns
SUCCESS, FAILED, or INVALID_STATE if out of sync
