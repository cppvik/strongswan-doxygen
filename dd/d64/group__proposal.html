)

Create a proposal from a string identifying the algorithms.

The string is in the same form as a in the ipsec.conf file. E.g.: aes128-sha2_256-modp2048 3des-md5 An additional '!' at the end of the string forces this proposal, without it the peer may choose another algorithm we support.
