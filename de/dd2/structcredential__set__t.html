Detailed Description

A set of credentials.

Contains private keys, shared keys and different kinds of certificates. Enumerators are used because queries might return multiple matches. Filter parameters restrict enumeration over specific items only. See credential_manager_t for an overview of the credential framework.

A credential set enumerator may not block the credential set, i.e. multiple threads must be able to hold multiple enumerators, as the credential manager is higly parallelized. The best way to achieve this is by using shared read locks for the enumerators only. Otherwise deadlocks will occur. The writing cache_cert() routine is called by the manager only if no enumerator is alive, so it is save to use a write lock there.
