static

This algorithm to linearize sections uses a bottom-first depth-first semantic, with an additional elimination step that removes all but the last occurrence of each section.

Consider this configuration:

conn A conn B also=A conn C also=A conn D also=C also=B

The linearization would yield D B A C A, which gets reduced to D B C A.

Ambiguous configurations are handled pragmatically.

Consider the following configuration:

conn A conn B conn C also=A also=B conn D also=B also=A conn E also=C also=D

It is ambiguous because D and C include the same two sections but in a different order.

The linearization would yield E D A B C B A which gets reduced to E D C B A.
